    1. Create `Dockerfile` with these exact contents:
       ```dockerfile
       FROM python:3.12-slim

       WORKDIR /app

       # Install dependencies first (layer caching)
       COPY requirements.txt .
       RUN pip install --no-cache-dir -r requirements.txt

       # Copy application code
       COPY solarman_logger/ solarman_logger/

       # Copy inverter definition profiles (required by ParameterParser)
       COPY custom_components/solarman/inverter_definitions/ inverter_definitions/

       # Default config path inside container
       ENV CONFIG_PATH=/config/config.yaml

       # Use exec form so Python is PID 1 and receives SIGTERM directly
       CMD ["python", "-m", "solarman_logger"]
       ```

       **IMPORTANT:** The `inverter_definitions/` directory must be copied because device YAML profiles reference it. The config.yaml `profile` field for each device will resolve relative to the config file's directory. Since the config is bind-mounted at `/config/config.yaml`, profile_dir will be `/config/`. So we need to either:
       - Copy inverter_definitions into the image and have the user reference `/app/inverter_definitions/` in their profile paths, OR
       - Document that users mount the definitions directory too

       The simpler approach: copy definitions into `/app/inverter_definitions/` in the image, and document that the `profile` field in config should use the filename only (e.g., `deye_micro.yaml`). The config loader resolves `profile_dir` from the config file's parent directory — so we need to handle this.

       Actually, looking at `config.py` line 114: `config_dir = str(abs_path.parent) + "/"` — the profile_dir is set to the config file's parent directory. If config is at `/config/config.yaml`, profiles would be looked for in `/config/`. This means inverter definitions need to be in `/config/` alongside the config file.

       **Revised approach:** Don't copy inverter_definitions into the image. Instead, document that the user should mount a directory containing both their `config.yaml` AND the `inverter_definitions/` folder, OR we update the Dockerfile to symlink/copy definitions to a known path and update config handling.

       **Simplest correct approach:** Copy inverter_definitions into the image at `/app/inverter_definitions/`, and add the profile directory path to the config YAML. But config.py always uses the config file's parent dir.

       **Final approach — keep it simple:** Mount the full config directory. The docker-compose.yml bind-mounts a host directory (e.g., `./config/`) to `/config/`. The user puts `config.yaml` AND the `inverter_definitions/` folder in that directory. This way `profile_dir` = `/config/` and profiles resolve correctly. The Dockerfile does NOT need to copy inverter_definitions. The config.example.yaml documents this.

    2. Final `Dockerfile`:
       ```dockerfile
       FROM python:3.12-slim

       WORKDIR /app

       # Install dependencies first (layer caching)
       COPY requirements.txt .
       RUN pip install --no-cache-dir -r requirements.txt

       # Copy application code only
       COPY solarman_logger/ solarman_logger/

       # Default config path inside container
       ENV CONFIG_PATH=/config/config.yaml

       # Stop grace period: Python needs SIGTERM as PID 1
       # exec form ensures python is PID 1
       STOPSIGNAL SIGTERM
       CMD ["python", "-m", "solarman_logger"]
       ```

    3. Create `docker-compose.yml`:
       ```yaml
       services:
         solarman-logger:
           build: .
           container_name: solarman-logger
           network_mode: host
           restart: unless-stopped
           stop_grace_period: 5s
           environment:
             - CONFIG_PATH=/config/config.yaml
           volumes:
             - ./config:/config:ro
       ```
