    **Fix 1 — Replace `multiprocessing.Event` with `asyncio.Event` (line 10, 77):**

    **Fix 2 — Replace recursive `_open_connection` with bounded retry loop (lines 213-225):**
    Replace the entire `_open_connection` method body with:
    ```python
    @throttle(0.2)
    async def _open_connection(self) -> None:
        max_attempts = 3
        for attempt in range(max_attempts):
            try:
                self._reader, self._writer = await asyncio.wait_for(asyncio.open_connection(self.host, self.port), self.timeout)
                self._keeper = create_task(self._keeper_loop())
                _LOGGER.debug(f"[{self.host}] {'Successful reconnection' if self._last_frame is not None else 'Successful connection'}!")
                return
            except Exception as e:
                if self._last_frame is None:
                    raise ConnectionError("Cannot open connection") from e
                _LOGGER.debug(f"[{self.host}] Connection attempt {attempt + 1}/{max_attempts} failed: {e}")
        raise ConnectionError(f"[{self.host}] Failed to connect after {max_attempts} attempts")
    ```
    Key changes from original:
    - `await self._open_connection()` recursive call → `for attempt in range(max_attempts)` loop
    - After max attempts exhausted → raise `ConnectionError` instead of infinite recursion
    - Removed the `if self._data_event.is_set(): await self._write(self._last_frame)` block (Fix 3)

    **Fix 3 — Remove last-frame replay after reconnection (per D-14):**
    The original had lines 217-219:
    ```python
    if self._data_event.is_set():
        _LOGGER.debug(f"[{self.host}] Successful reconnection! Data expected. Will retry the last request")
        await self._write(self._last_frame)
    ```
    These are removed entirely in Fix 2's replacement. After reconnection, the client simply logs success and returns. The next scheduled poll cycle will issue fresh requests.

    **DO NOT change any other methods** — `_keeper_loop`, `_send_receive_frame`, `execute`, `close`, `_parse_adu_*` remain untouched.
